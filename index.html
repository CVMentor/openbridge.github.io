Welcome To Openbridge

Overview

This documentation is intended to get you the basics of running Openbridge applications. We’ll cover everything you need to know, from authentication, to data delivery, to systems setup. All code examples will be stored and documented in our public repositories. Feel free to fork, clone, and improve the sampels code and guides.

Openbridge Platform

Data Pipelines: The Openbridge Data Pipeline include two specific resources for delivery of data into our system;

  • Streaming API
  • Delivering Data

Data Bridges: Data Bridges are considered “outbound” data services. They are adapters that are built for “fetching” (connecting, processing and storing data) from a third party data source.

Data Processing: Processing Service is a distributed real-time computation system for processing fast, large streams of data

Data Warehouse: Warehouse Services is a fast, fully-managed, petabyte-scale data warehouse service.

Data Replay: Real-time in-memory cache and long term data retention

Expert Services: Development, consulting or subject matter expertise for success.

Guides

The following is a limited collection of guides to help provide context on how to use the Openbridge system.

Adobe Analytics http://adobe.com

Adobe Analytics provides one of the leading web analytics solutions on the market. We have detailed two methods of extracting data from Adobe to Openbridge.

  • Adobe Clickstream Feeds (ACF): We provided hints, considerations, samples and steps to follow for accepting Adobe Clickstream feeds.

  • Adobe Datawarehouse Feeds (ADF): Documentation of setting up ADF to Openbridge will be live shortly. We are putting some finishing touches them. Contact support for more information on setting up Adobe Datawarehouse feeds.

Kenshoo http://kenshoo.com

Kenshoo provides the ability to push feeds directly to Openbridge. Please contact your Kenshoo account rep to request feeds be configured.

  • Kensoo Feeds: Notes on delivery of paid social data from Kenshoo to Openbridge.
